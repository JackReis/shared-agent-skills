# Cron Prompt Overwrite vs Append — 2026-06-12 Incident

## The Bug

The **Daily Briefing Assembly** cron job (`6dd268a3c89f`, runs daily at 04:00 CDT) overwrote `=notes/claude/coordination/inbox-fleet.md` — a 311-line append-only coordination log — with a 12-line auto-generated stub.

## Root Cause

The cron job prompt step 6 said:
> "6. **Notify Fleet**: Post a summary to `=notes/claude/coordination/inbox-fleet.md`."

The LLM agent interpreted "Post a summary to <file>" as `write_file(path=..., content=...)` — a full overwrite. The file was replaced with:
```
# Fleet Inbox
## 📥 Incoming Updates
- **2026-06-12 Daily Note**: Created...
*Auto-generated by Hermes. Do not edit directly.*
```

All 311 lines of prior fleet coordination history were destroyed.

## Evidence

From the cron session dump at `~/.hermes/sessions/session_cron_6dd268a3c89f_20260612_043112.json`:
- Line 3207: "The `inbox-fleet.md` file does not exist in the expected location. Creating it..."
- Line 3218: `write_file` call with 422 bytes — a stub, not an append
- Line 3227: `"bytes_written": 422` confirmed the overwrite

## Recovery

The file was restored from git HEAD (`git checkout HEAD -- claude/coordination/inbox-fleet.md`), the stub was quarantined to `claude/archive/inbox-fleet-overwrite-20260612/`, and its 2 novel entries were re-appended. Zero information lost.

## Fix Applied

The cron job prompt was updated to explicitly say:
> "6. **Notify Fleet (APPEND ONLY — NEVER OVERWRITE)**: APPEND a summary to `=notes/claude/coordination/inbox-fleet.md`. Use `patch(mode='replace', old_string='*Auto-generated by Hermes. Do not edit directly.*', new_string='')` to remove the auto-gen footer if present, then APPEND your new entry. NEVER use `write_file` on this file — it is an append-only log."

## Class-Level Lesson

Any cron job prompt that says "post to X file" or "notify via Y file" without explicit APPEND instructions will cause the LLM agent to use `write_file` and destroy the file's history. This applies to:
- Fleet inboxes (`inbox-fleet.md`)
- Coordination ledgers (`LEDGER.md`)
- Daily notes (though these are usually created fresh)
- Any append-only log file

The fix is always the same: say "APPEND ONLY — NEVER OVERWRITE" in the prompt, and give concrete instructions for how to append (patch to remove footers, then append with specific format).
